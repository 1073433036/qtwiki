Typos and Corrections posted before July 2011

All typos and corrections listed here should be fixed in the new version [qt.nokia.com] of the course material posted on July 3., 2011. If you find that they are not, please move them back [developer.qt.nokia.com].

General

  • fixed Please rebuild the zip archive excluding the (in)famous __MACOSX directory and the .DS_Store files. –
  • Still, other .zip files do have those files. Please rebuild them all :-)

L1

  • Slide 5, typo: QObject *o = new QPustButton; : Should be "QPushButton" – fixed in release of Dec 23, 2010
  • Slide 3, wrong double quotation marks (should be " instead of " when opening)
  • Slide 3, typo: original -> originally
  • Slide 3, typo: etcetera -> etc., or et cetera [en.wikipedia.org]
  • Slide 4, "Qt is built from modules" -> "Qt is made up of modules"

L2

  • Slide 36 (only valid for the slides with notes). typo in notes: "Signals re defined in the…" should be "Signals are defined in the…" – fixed in release of Dec 23, 2010
  • Slide 32: the lines connect the wrong buttons to the slots
  • Slide 36: the "emit" keyword actually is #defined as nothing. Perhaps a note there?

L3

  • Slide 40: (Which is similar to L2 slide 8 / 9): why "mocs" and "uic"? Why singular forms at all?

L4

  • Slide 4: don't mention toAscii(), or add a warning: it doens't convert to ASCII as its name suggests :|
  • Slide 11: wrong; QString uses toAscii() and fromAscii() to handle the conversions, which may not be ASCII at all.
  • QLatin1String is never mentioned (which is instead the recommended way of putting Latin1 literals); the QString(const char *) ctor should never be used instead…
  • Slide 14: "Cheap to pass as const references" actually it's nonsense, because EVERYTHING is cheap to pass as a const-ref. Implicitly shared classes are cheap to pass and return by value (passing by const-ref is even cheaper, ok, but it's misleading), like the aforementioned QString::split method does.
  • Slide 18: calling length() at every loop is more expensive than just calling it once at storing its value
  • Slide 18: it's important to stress that operator[] on a non-const Q(String)List detaches! (i.e. makes a deep copy)
  • Slide 19: mention that QMap is sorted by key (unlike QHash, which therefore can be faster)
  • Slide 21: replace while(list.length()) with while(!list.isEmpty()) in the first 2 examples
  • Slide 21: stress on the fact that remove() simply removes, take() removes+returns
  • Slide 21: weird font on "removeOne" :-)
  • Slide 22: QList::value() does not return a reference but a value
  • Slide 22: again, good place to mention the operator[] detach
  • Slide 24: STL and Qt naming of iterators are swapped. STL: const_iterator; Qt: ConstIterator. It's better to use the STL naming in the example, and fix the green leaf on the right
  • Slide 27: Discuss QList internals (an array of pointer to the actual elements, or, if sizeof(element)<=sizeof(void*), the element themselves are stored in the array), as well as QLinkedList (double linked list) and QVector ones (array of elements). This helps a lot when it comes to understand the pro/cons of each class.
  • Slide 27: "Fast when using iterators" is wrong; all iterators on list structures are fast
  • Slide 28: QStack is not based at all on QList (actually, it subclasses QVector. The stack elements must be contiguous in memory)
  • Slide 28: QSet is not based at all on QList (it's a simple wrapper for QHash<T, dummyvalue>. Check http://qt.gitorious.com/qt/qt/blobs/master/src/corelib/tools/qset.h#line53 )
  • Slide 35: It's a better idea to use ^ rather than + to hash.
  • Slide 38: sizeof(int) => sizeof(long)? Int is 32 bit even under IA64…
  • Slide 49: Why "C:\\" and not "C:/", immediately telling people that Qt uses forward slashes everywhere?
  • Slide 49: Is it worth to mention QDesktopServices::storageLocation?

L5

  • Slide 74: typo: QFontMetric => QFontMetrics
  • Slide 83: typo: paint to it => paint on it

L6

  • Slide 4: Widgets CAN overlap (but usually they don't, because they're inside a layout); widgets are repainted when it's necessary, not only when update() is called, exactly like QGraphicsItems.
  • Slide 6: typo: "moving about" => "moving around"

  • Slide 7: "import Qt 4.7" has been deprecated in favour of "import QtQuick 1.0"
  • Slide 11 (notes for slide): Typo: "There are scooping rules" must be "There are scoping rules"
  • Slide 16 (notes for slide): Typo: "makes the them feel limiting" -> "makes them feel limiting"
  • Slide 24: typo: "feasible", not "feasable"
  • Slide 26: code frame has text "1" on center, need to delete this
  • Slide 26: typo: "Instatiate" must be "Instantiate"
  • Slide 27 (notes for slide): Typo: "to make is possible" -> "to make it possible"
  • Slide 28 (notes for slide): Typo: "tree states" -> "three states"
  • Slide 29: a sidenote slipped in?
  • Slide 29 (notes for slide): Typo: "using PropertyChange elements" -> "using PropertyChanges elements"
  • Slide 34: "QGraphicsScene* scene = …" must be "QGraphicsScene *scene = …" according to coding style
  • Slide 35 (notes for slide): Typo: "If you do now have" -> "If you do not have"
  • Slide 37: "NOKIA" logotype overlapped by frame
  • Page 3: Figure with rectangles should be rotated 90 degrees clockwise for the corresponding to QML code in this document.

L10
