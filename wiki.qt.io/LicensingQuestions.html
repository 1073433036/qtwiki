Licensing Questions

Licensing can be a mystery sometimes. You can add your suggestions for which topics we should cover here.

Before you do so, please, take a look at this blog (Licensing blog) and the video (Licensing video) to check if your question is already answered there.

Question: Closed-source application scenario

Consider the following hypothetical scenario:

Someone writes a closed-source application using Qt SDK and Qt Creator (which they have downloaded by accepting the LGPL choice and licence) for Windows. They do not change the libraries. They link dynamically (shared) against these libraries (.dll files).

Subcases:

  1. They do not provide (distribute) the dll-files but they only provide the executable file ".exe".
  2. They provide (distribute via web) all needed files for the application to run (dll-files and .exe).

I would like Nokia’s legal thesis on the following:

What are their obligations to be consistent with the LGPL choice they made when downloading and installing Qt SDK and Qt Creator? Special point of interest (for each of the two subcases): do they have to provide (distribute) the, unchanged, source code of the dll-files they used (and in the subcase 2 included these dll-files)?

Question: Qt needed dll-files licenses?

What are the licences of the following dll-files (which are needed for Qt applications):

mingwm10.dll

  • libgcc_s_dw2-1.dll

and their obligations? These files are installed with the installation of Qt SDK + Qt Creator and needed for the executable file to run.

If Nokia’s answer is that source code must be provided (distributed), could (from a technical and legal point of view) Nokia provide Qt’s dll-files (QtCore4.dll, QtGui4.dll, etc) and other needed files (mingwm10.dll, libgcc_s_dw2-1.dll) for separate download (without having to download the complete SDK) so that the distribution of the executable file from a distributor could be accompanied with the links from Nokia’s servers for all the needed dll-files without the distributor would have to provide source code of the dll-files as well?

Question: 3rd party stuff

What I am missing is a nice overview of which 3rd party tools, plugins and drivers are available under which license.

In my case I need information under which license the Oracle Database driver can be used. I've got some information from Nokia guys on IRC, but I was unable to find a solid answer in the official documentation.
