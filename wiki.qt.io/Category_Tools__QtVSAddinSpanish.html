Qt Visual Studio Add-in

Obtén el código

El código fuente del add-in se encuentra alojado en gitorious.

Para crear un clon del repositorio ejecute el siguiente comando 

git clone git://gitorious.org/qt-labs/vstools.git<code>
 
== Instrucciones para compilar ==
 
Asegúrese de tener instalado el siguiente software.
 
* Visual Studio 2005 o superior
* Perl
* Qt 4.7.0 o superior
 
Abra un shell de comandos con el entorno de compilación de Qt (Visual Studio build, 32 bit) y utiliza el comando '''cd''' para posicionarse en el directorio donde esta ubicado el código fuente, luego ejecute los siguientes comandos
cd Qt4VS2003 createUserFiles.cmd
Esto establece las rutas de referencia y la configuración de depuración.
cd ComWrappers\qmakewrapper

qmake nmake

cd ..devenv qtappwrapper2008.sln /useenv /Build release
Establezca el número de qtappwrapper2008.sln según la versión de Visual Studio que este utilizando. Existen versiones del add-in para cada versión soportada de Visual Studio.
cd ..qmake nmake
Esto genera el editor externo QRC que se lanza desde dentro de Visual Studio. Ahora estamos listos para abrir la solución principal que contiene el actual add-in.
start Qt4VS2003\Qt4VSAddin2008.sln</code>

Ahora debería ser capaz de generar la solución. Antes de intentar hacer funcionar el add-in recién generado, lea la siguiente sección.

Ejecutar y Depurar

Para correr y depurar la versión de desarrollo del add-in de Qt para Visual Studio, necesitamos nuevamente hacer algunos ajustes de configuración. Tenga en cuenta que puede tener la versión del add-in instalada, también puede cambiar entre dos instalaciones como veremos pronto.

Instalar el Archivo de Descripción del Add-In

Abra el directorio donde se encuentra guardada su configuración personalizada (settings) de Visual Studio. Usualmente es algo como "USERPROFILE\Documents\Visual Studio 2008". Vamos a usar este directorio a modo de ejemplo. También vamos a asumir que el repositorio del add-in se encuentra ubicado en c:.

Ejecute los siguientes comandos: 

cd "USERPROFILEStudio 2008"
md Addins
cd Addins
copy c:4VS2003\Qt4VSAddin\Qt4VSAddin.AddIn
.

Qt4VSAddin.AddIn es el archivo de descripción y ahora Visual Studio puede encontrarlo. Para otras versiones de Visual Studio diferentes de la 2008, debe ajustar la etiqueta (tag) Version en el archivo Qt4VSAddin.Addin.

La versión release instalada de el add-in se encuentra desactivada y no puede ser cargada por Visual Studio. Para restaurar el viejo estado, renombre la extensión del archivo Qt4VSAddin.AddIn por algún valor temporal como por ejemplo Qt4VSAddin.AddIn.foo. Esto nos asegura que sera ignorado por Visual Studio y el add-in original sera cargado.

Correr la versión de desarrollo del Add-in

Ahora que el archivo .Addin esta en su lugar, estamos listos para probar las versión de desarrollo del add-in. Abra la solución (e.g. Qt4VSAddin2008.sln) y presione F5.

Una nueva instancia de Visual Studio se iniciara. Llamaremos a esta instancia VS2 y la instancia de Visual Studio donde la solución con el add-is se carga como VS1.

El add-id no esta aun cargado en VS2. Para cárgalo, seleccione "Tools/Add-in Manager…" ,haga click en el primer checkbox en "Qt4 Add-in Development Version" y presione OK.

Ahora tiene una entrada de Menu "Qt" en la barra de menu de Visual Studio, lo que significa que el add-in ha sido cargado exitosamente.
