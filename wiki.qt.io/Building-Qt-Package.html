Qt5 offline installer creation instructions

Required tools:

  • Python
  • Git
  • 7z
  • chrpath (Linux)
  • otool (Mac)

Required tools for building Qt 5:

  • Perl(note: need to be active perl)
  • Python 2.x(needed for webkit build)
  • Python win32api
  • Ruby(needed for webkit build)

Required 3rd party libraries for building Qt 5(needed for webkit):
– icu .49

Setup environment correct for Windows: For this purpose this following script can be used if the path's to the tools are replaced with the actual ones: 

call "VS100COMNTOOLS.bat" x86
set PATH=PATH;C:lt;path-to-7zip>
set PATH=PATH;C:lt;path-to-git>
set PATH=PATH;C:lt;path-to-python>
set PATH=PATH;C:lt;path-to-perl>
set PATH=PATH;C:lt;path-to-ruby>
set PATH=PATH;C:lt;path-to-icu>
set INCLUDE=INCLUDE;C:lt;path-to-icu>
set LIB=LIB;C:lt;path-to-icu>
rem If you want to open a command line use the cmd command
call cmd.exe
  • Step 1: Fetch packaging scripts and init Qt sources 
git clone git://code.qt.io/qtsdk/qtsdk.git
cd qtsdk
perl init-repository -f -q
  • Step 2: Create source packages 
release-tools/mksrc.sh -v 5.0.0 -i qt-creator -m

' Explanations' "-i" to ignore given submodule from the archive "-m" to generate submodularised source archives as well

  • Step 2.1: Copy source archives into e.g. network disk for later usage (installer creation) 
scp -r single user@localhost:/var/www/html/qt/5.0.0_opensource/src
scp -r submodules_tar user@localhost:/var/www/html/qt/5.0.0_opensource/src
scp -r submodules_zip user@localhost:/var/www/html/qt/5.0.0_opensource/src
  • Step 3: Create configure.exe (only Windows) and put it into source arhive 
mkdir qt5_configure
cd qt5_configure
python -u ../release-tools/helpers/create_configure_exe.py src_url=qt-everywhere-opensource-src-5.0.0.zip
—mdl_url=qtbase-opensource-src-5.0.0.zip
  • Step 3.1: Put back the modified source archive 
scp.exe qt-everywhere-opensource-src-5.0.0.zip
user@localhost:/var/www/html/qt/5.0.0_opensource/src/single
scp.exe qtbase-opensource-src-5.0.0.zip
user@localhost:/var/www/html/qt/5.0.0_opensource/src/submodules_zip
  • Step 4: Create binary packages 
python release-tools/mkqt5bld.py —src-url=qt-everywhere-opensource-src-5.0.0.tar.gz —jobs=16 -i qt-creator

' Explanations' "—src-url" where to fetch the src package, can be remote or local "—jobs" i.e. the wanted thread count for build "-i" exclude <module> from the build "-m jom" use Jom (Windows only)

  • Step 4.1: Copy generated binary archives into network disk (for installer creation) 
scp -r module_archives/*.7z user@localhost:/var/www/html/qt/5.0.0_opensource/linux_gcc_64_ubuntu1110
  • Current configure options used by mkqt5bld.py:
    • Linux: -opensource -confirm-license -debug-and-release -release -nomake tests -nomake examples -qt-zlib -qt-libjpeg -qt-libpng -L /<build_dir>/icu_install/lib -I /<build_dir>/icu_install/include -no-gtkstyle -qt-xcb -prefix /<build_dir>/_PADDING_ -R /<build_dir>/_PADDING_
    • Windows: -opensource -confirm-license -debug-and-release -release -nomake tests -nomake examples -qt-zlib -qt-libjpeg -qt-libpng -angle
    • Mac: -opensource -confirm-license -debug-and-release -release -nomake tests -nomake examples -qt-zlib -qt-libjpeg -qt-libpng -no-pkg-config -platform macx-clang -prefix /<build_dir>/_PADDING_
    • Padding is used to accommodate long paths for RPath
  • Step 5: Create qt-creator binary package

unknown how it works at the moment, TODO

  • Step 6: Create installers (on Linux you need to install "chrpath") 
python release-tools/create_installer.py -f linux_x86_64_u1110_offline —offline —archive-base-url=http://localhost/
  • Explanations

By default the script will look for configuration files under /release-tools/configurations "—configurations-dir" to define other directory where to look for configuration file " –f" define configuration file to be used (under /release-tools/*configurations*) e.g. "linux_x86_64_u1110_offline" "—offline" for offline installer "—archive-base-url" define location where from to look for packages for installer, this is a URL where you have all the binaries with a certain dir structure (i.e. the ones created in steps 2-4) Add "—devmode" if you wan't to build Installer-Framework from sources. Otherwise the script will try to fetch the pre-built Installer-Framework libraries and use them For other options see: TODO
