Tagging

Durch Hinzufügen von Tags im DevNet kann man anderen Usern helfen, thematisch Informationen zu finden, die zu ihren Anforderungen opassen. Deswegen bekommt man Punkte für das "Taggen" (Hinzufügen von Tags). Taggen ist für das Forum, FAQ Einträge, statusche seiten und Wiki Einträge freigeschaltet.

Nachfolgend eine Liste, welche Tags derzeit genutzt werden:

Das Entfernen von Tags ist für "Hobby Entomologist" und höhere Ränge freigeschaltet.

Tags mergen ist derzeit nicht im Frontend implementiert . Tags können nur von Admins gemergt werden.

Allgemeine Regeln für Tags:

  • Bitte nur die Einzahl verwenden
    • Beispiel: "nokia phone" anstelle von "nokia phones"
  • Tags unterstützen Leerzeichen, also sollten sie nicht entfernt werden
    • Beispiel: Benutzt "qt creator" undnicht "qtcreator"
  • Bitte sprechende Tags verwenden:
    • Beispiel: Nicht mit "qt" taggen - wir sind innerhalb eines Qt Forums, das ist "normal"
  • Inhalt, der nicht Englisch ist, sollte mit folgendem Tag gekennzeichnet werden: "language:code"
    • Beispiel: language:ru, language:es, language:de usw.
  • Keine Tags durch verwendung von Sysonymen duplizieren (merging wird das beheben)
    • Beispiel: "visual studio", "msvs", "msvc"
  • Fehler relevanter Inhakt wird manchmal mit einem speziellen Pattern getaggt:
    • Beispiel: qtwebsite-70

Folgende Tags werden automatisch wieder gelöscht, wenn sie gesetzt werden:

  • "faq" - in der sektion "FAQ"
  • "wiki" - innerhalb des Wiki
  • "devnet" - Eigentlich kein Wert (wir sind im Devnet) und wegen Fehlbenutzung
  • "beta" - Wenn irgendetwas als beta gekennzeichnet werden soll, muss eine Versionsnummer mit angegeben werden

Kandidaten:

  • "question" – Nutzen unklar, sehr häufig benutzt
  • "please rename" – innerhalb des Wiki, aber bis jetzt gibt es keine Namenskonventionen

Macht "sub-tagging" Sinn?

Nachfolgend ein kurzes Beispiel das QtK im Forum aufgebracht hat: Inhalt der Project Elf relevant ist, kann mit folgenden Tags Kombinationen der folgenden Wörtern "elf" und "project" markiert werden. Manche Kombinationen sind besser als andere:

  • "elf, project" - ok
  • "project, project elf" - ok
  • "elf, project elf" - nicht so gut
  • "elf, project, project elf" - nicht so gut

Nachdem "elf" ein Synonym für project elf ist, ist eine Alternative von"elf" und "project elf" föllig ausreichend. Wenn das Tag "project" nicht hinzugefügt wird, wird die Seite nicht mit anderen Projekten zusammen gelistet.
