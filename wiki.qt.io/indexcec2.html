Etiquetando

Mediante la adición de etiquetas (tags) al contenido de DevNet podemos ayudar a otros usuarios a encontrar la información por temas mas cercanos a sus necesidades, es por eso que es posible conseguir puntos por etiquetar. El etiquetado está habilitado para los mensajes del foro, las entradas FAQ, las páginas estáticas y las páginas de la wiki.

Aquí puedes ver una lista de las etiquetas mas usadas comúnmente

La función de remover etiquetas esta disponibles para usuarios con rango de "Hobby Entomologist" o superior.

La fusión de etiquetas aun no esta implementada en el front end. Fusionar etiquetas por ahora solo puede ser usada por Admins.

Guía general para etiquetar:

  • No use palabras en plural, solo singulares
    • Ejemplo: etiqueta con "nokia phone" no "nokia phones"
  • Las etiquetas soportan espacios
    • Ejemplo: etiqueta con "qt creator" no "qtcreator"
  • Use etiquetas signnificativas
    • Ejemplo: no etiquete con "qt" - este es un foro de Qt
  • Contenido que no este en ingles debe ser etiquetado con "language:code"
    • Ejemplo: language:ru, language:es, language:de etc.
  • No duplique etiquetas existentes usando sinonimos de ellas (la opción de fusión de etiquetas resolverá esto)
    • Ejemplo: "visual studio", "msvs", "msvc"
  • Contenido relacionado con reportes de errores (bug entries) siguen un patrón especial
    • Ejemplo: qtwebsite-70

Tiene sentido el "sub etiquetado"?

Aqui un pequeño ejemplo inspirado por QtK en el foro: Contenido relacionado a Project Elf puede ser marcados con una combinación de etiquetas a partir de las palabras "elf" y "project". Al parecer algunas combinaciones son mejores que otras:

  • "elf, project" - ok
  • "project, project elf" - ok
  • "elf, project elf" - no tan buena
  • "elf, project, project elf" - no tan buena

"Elf" probablemente se mantendrá sólo como un sinónimo de project elf, una de las alternativas "elf" y "project elf" será suficiente. Cuando la etiqueta de "project" no se agrega a el contenido este no sera mostrado en otras paginas de proyectos.
